规范与阿里巴巴Java开发手册

Jan 5, 2019



如果把《阿里巴巴Java开发手册》当成一本书的话，那这肯定是我看过次数最多的书。
记得当时第一次阅读这个手册的时候，是觉得越看越吃惊，真的没想到有人能把经验问题总结的这么好，而且已经系统的做成手册了。里面指出的不少规范问题，也时常出现在我过去的几年编码经历中，这就更让我从内心深处产生了共鸣，对此当然也奉若珍宝。即使已经通读过了好几遍，还是会偶尔翻出来看看，加深印象。
以前在面试的过程中，曾被问过这样的一个问题：
::java中的基本类型int和包装类型Integer，在使用过程中该如何选择？::
很可惜，当时没有给出一个好的答案。在那之后，对此也有过思考，但是一直没有找到一个系统的答案。很庆幸，它很好的回答了我。

代码规范一直是我比较看中的技能，会写功能跟会写代码是两回事，多少人的代码在完成数周以后，连自己都看不懂。在面试求职者的过程中，这块知识也是我经常要问的。招一个技术好但没有规范的程序员，还不如招一个技术一般但有着遵代码规范的程序员。至少后者会让你想优化重构的时候，有的放矢。

在19年1月份的学习计划中，又习惯的把它加了进去。
恰好在阿里云上找到这个认证考试：Apsara Clouder基础技能认证：阿里巴巴编码规范)。
很有意思的一个认证，报名费九块九。
