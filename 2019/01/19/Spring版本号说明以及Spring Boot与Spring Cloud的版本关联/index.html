Spring版本号说明以及Spring Boot与Spring Cloud的版本关联

1月 19 2019

一篇资源整合形式的文章，去了解关于Spring版本的那些事。

Spring的版本说明

参考: What is the difference between Spring’s GA, RC and M2 releases? - Stack Overflow

  • GA(General availability)：一般可用性（发布）;应该非常稳定，功能齐全
  • RC(Release candidate)：发布候选,可能功能完整，应该非常稳定 - 问题应该是相对罕见和轻微的，但值得报告，试图修复它们以便发布。
  • M(Milestone build): 里程碑建设,功能可能不完整,稳定性一般可能仍有问题。

Spring Boot与Spring Cloud的版本联系

翻译至：Spring Cloud Release Trains

Spring Cloud是一个由独立项目组成的总体项目，原则上具有不同的发布节奏。为了管理项目组合，它发布了一个BOM(材料清单)，包含了一组对单个项目的依赖关系(参见下面)。Release Trains有名称而不是版本，以避免与子项目的版本混淆。这些名称是按照伦敦地铁站的名称字母排序（所以你可以按时间顺序排序）（“Angel”是第一个版本，“Brixton”是第二个版本）.当各个项目的发布点累积到一个临界值时，或者其中一个项目存在影响到每个使用者的严重的bug,则Release Trains将推出名称以“.SRX”结尾的版本，其中X是一个数字。

  • Angel版本系列基于Spring Boot 1.2.x构建，在某些方面与Spring Boot 1.3.x不兼容。 Brixton构建于Spring Boot 1.3.x之上，与1.2.x同样不兼容。一些基于Angel构建的库和大多数应用程序都可以在Brixton上正常运行，但是在使用spring-cloud-security 1.0.x的OAuth2功能的任何地方都需要进行更改（它们大部分都转移到1.3.0中的Spring Boot）。
  • 使用依赖关系管理工具来控制版本。如果您正在使用Maven，请记住声明的第一个版本会优先使用，因此请按顺序声明bom，第一个通常是最新的（例如，如果您想使用带有Brixton.RELEASE的Spring Boot 1.3.6，请首先放置Boot的BOM）。如果您使用Spring依赖关系管理插件，则同样的规则适用于Gradle。
    注意: 版本系列包含spring-cloud-dependencies和spring-cloud-starter-parent。您可以像使用spring-boot-starter-parent一样使用parent（如果您使用的是Maven）。如果您只需要依赖关系管理，那么“dependencies”版本是仅限BOM的版本（它只包含dependency management,没有插件声明或直接引用Spring或Spring Boot）。如果您使用的是Spring Boot parent POM，则可以使用Spring Cloud中的BOM。相反的情况并非如此：使用Cloud parent, 并且使用Boot BOM来更改Spring Boot及其依赖项的版本,是无法实现或者至少是不可靠的。
